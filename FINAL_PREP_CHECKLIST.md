## üí° KEY CONCEPTS TO REMEMBER

### The Big Picture
All 5 assignments demonstrate **concurrent programming** - running multiple tasks simultaneously to make programs faster and more efficient.

### Assignment Quick Summary

**Assignment 1**: Multithreading makes things faster (10 threads = 10x speed)

**Assignment 2**: Two ways to make code thread-safe (lock everything vs smart collections)

**Assignment 3**: Board game showing good design patterns (MVC, Strategy)

**Assignment 4**: Prevent 3 problems (deadlock, livelock, starvation) with 3 solutions (semaphore, timeout, fairness)

**Assignment 5**: Producers and consumers coordinated automatically (BlockingQueue is magic)

### Most Important Technical Terms

1. **Thread**: A worker in your program
2. **Synchronized**: Only one thread at a time (like a lock)
3. **Immutable**: Can't be changed (automatically thread-safe)
4. **Deadlock**: Everyone stuck waiting forever
5. **Semaphore**: Tickets controlling access
6. **BlockingQueue**: Smart buffer that handles waiting automatically
